# PLAN Stage

You are an autonomous coding agent. Your job is to create a detailed implementation plan.

## Context
- Read `issue_body.md` for the task description and acceptance criteria
- Read `repo_summary.md` for the repo structure, CI setup, and contributing guidelines

## Repo Learnings
- Read `learnings.md` (from the run directory) — it contains important repo-specific
  conventions, gotchas, and patterns discovered from previous work.
- Follow these learnings. They take priority over general assumptions.

## Instructions
1. Identify which files need to be created or modified
2. Trace the user path — how does a user reach this feature?
3. Identify what tests are needed
4. Check if a feature flag is needed
5. Write your plan to `plan.md` in the run directory

## Output
Write `plan.md` with:
- Summary of the change
- List of files to create/modify (with brief description of each change)
- Test strategy
- UI wiring needed (if any)
- Risks or open questions
